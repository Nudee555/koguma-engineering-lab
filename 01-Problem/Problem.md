# Problem

## Scenario

An academic institution needs a simple enrollment system that allows students to register for courses. The system must verify the student's identity, check course availability, record the enrollment, and return a successful confirmation. To visualize the design before writing code, a UML Class Diagram and a UML Sequence Diagram were created to model the system structure and the interaction between its components.

## Why It Matters

Designing a system before writing code reduces mistakes, improves communication among developers, and creates a shared understanding of how the application should behave. UML diagrams help engineers identify relationships, system responsibilities, and workflow before implementation begins.

## Business Impact

A well-designed enrollment system reduces registration errors, prevents duplicate enrollments, improves administrative efficiency, and provides students with a faster registration experience. Modeling the system before development also reduces software defects, lowers maintenance costs, and shortens future development time.
